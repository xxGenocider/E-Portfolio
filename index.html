Shortest Path Problems

This is one of the topics I enjoyed the most in this course. It is quite easy but at the same time challenging. The study guide also supplies useful real-life applications of the topic. This topic teaches you how to figure out the shortest path based on a graph. The graph has the nodes (location) and the edges (path) with their weight (assigned value).

One algorithm was introduced in this topic, the Dijkstra algorithm. The algorithm is the most used in solving shortest path problems. One of the important features of the Dijkstra algorithm is its efficiency. The algorithm is very efficient in terms of the number of comparisons that it makes. Additionally, the algorithm is also very fast in terms of the time required to sort the sequence. Consequently, the Dijkstra algorithm is widely used in real-world applications.

Connectivity

This topic teaches how to figure out if a graph is strongly connected or weakly connected. The graph is said to be strongly connected if there is a path between a and b. The graph is said to be weakly connected if there is an undirected path between vertices in an undirected graph.

This topic is an important tool that can be used in a variety of different applications. For example, connectivity is often used to determine the coordination of activities in a large, distributed system. In a large system, it is important to determine which tasks need to be coordinated in order to ensure that they are completed in a timely manner. Connectivity also plays a role in the communication between components in a system. By understanding the connectivity of a system, we can better understand the interactions between its individual elements.

Euler and Hamilton Paths

I find this topic interesting and the most challenging out of all topics dealing with graphs. Eulerian paths and circuit deals with edges while Hamilton paths and circuit deals with vertices.

In graph theory, the Euler path (Eulerian trail) is a shortest path between two vertices in a connected Graph. The Euler path can be computed by following an edge only once and does not intersect any other edges along the way.

Euler cycle is similar to Euler path, but the difference is that for a network to be called a Eulerian (Eulerian cycle) it needs to start and end in the same vertex and use all the edges once.

Hamiltonian paths are paths that visits every vertex in a network exactly once. This is useful knowledge when planning for a trip. You can use Hamiltonian path to determine the path you take to visits all the places you want efficiently.

Hamiltonian cycle is similar to Hamiltonian paths. However, in Hamiltonian cycle, you must start and ends in the same vertex.
